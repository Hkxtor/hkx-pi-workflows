# HKX Web Design Quality

Use this rule when designing or editing meaningful web UI surfaces.

Do not ship generic template output. A finished screen should look intentional, product-specific, and believable in a real product screenshot.

Avoid:

- Stock centered hero sections, gradient blobs, and generic CTAs.
- Default card grids, dashboards, or component-library layouts with no point of view.
- Safe gray-on-white styling with one decorative accent color.
- Uniform spacing, radius, shadows, and emphasis across every component.
- Unmodified library defaults presented as final design.

Before writing UI code, choose a concrete visual direction for the product. "Clean" or "modern" is not enough. Decide the hierarchy, palette, typography, layout rhythm, and interaction feel first.

Every substantial surface should show several of these qualities:

- Clear hierarchy through scale, contrast, grouping, and information priority.
- Intentional spacing rhythm instead of equal padding everywhere.
- Typography chosen for the product voice, with deliberate weight and size changes.
- Color used semantically for state, priority, and mood.
- Layout composition with structure: editorial, bento, split-pane, data-dense, immersive, or another explicit direction.
- Hover, focus, active, loading, empty, and error states that feel designed.
- Depth, layering, motion, texture, or atmosphere only when it supports comprehension.
- Data visualizations styled as part of the system, not dropped in as defaults.

Default to the product's needs, not dark mode, framework defaults, or trend imitation. If both light and dark themes exist, both must feel intentionally designed.
